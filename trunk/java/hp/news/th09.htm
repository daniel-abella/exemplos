Programa concorrente (estende Thread)

Um programa concorrente executa mais de uma tarefa em um mesmo instante de tempo. Por exemplo, a classe Concorrente, declarada abaixo, executa duas tarefas concorrentemente. Cada uma destas tarefas é executada pelo método tarefa(int). Ou seja, este programa irá executar concorrentemente duas chamadas a este método. Uma delas irá receber como argumento o valor 1 e outra o valor 2, conforme explicado abaixo.

A classe Concorrente estende a classe Thread e sobrepõe o método run(). Quando esta classe é executada, o método main cria uma instância de Concorrente e envia a mensagem start() para a instância criada. Isto faz com que uma novo thread seja criado. Este novo thread inicia a sua execução pelo método run() que executa uma chamada tarefa(1).

Após a mensagem start() temos dois threads em execução concorrentemente: aquele criado pela máquina virtual Java executando o método main e aquele explicitamente criado pelo programa e executando o método run(). Cada um deles é executado concorrentemente e, dessa forma, o primeiro deles não fica bloqueado na chamada a start() aguardando o fim da execução do método run(). Prossegue executando a chamada tarefa(2). Portanto, tarefa(1) e tarefa(2) serão executadas concorrentemente. Em conseqüência, impressões que se iniciam com 1 e 2 deverão se intercalar na saída, mostrando que há duas sentenças for em execução concorrentemente. 

public class Concorrente extends Thread {
    
    public void tarefa(int prefixo) {
        for (int i = 0; i < 100; i++) {
            System.out.println(prefixo + " " + i);
        }
    }
    public void run() {
        tarefa(1);
    }

    public static void main(String[] args) {
        Concorrente cc = new Concorrente();
        cc.start();
        cc.tarefa(2);
    }
}
