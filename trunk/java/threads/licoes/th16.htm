Escalonador de threads

A especificação da máquina virtual Java não exige a implementação de time-slicing. Nesta abordagem, cada thread recebe uma quantidade de tempo após o qual o thread retorna ao estado executável e permite que outro thread tenha a oportunidade de ser executado, pelo mesmo período de tempo. Esta é uma proposta "comunista" de execução, onde todos os threads são tratados e atendidos igualmente.

Embora time-slicing seja uma implementação típica, ela não é exigida e, em conseqüência, alguma implementação da máquina virtual Java pode fazer com que um thread execute até que o método run() termine e, só então, oferecer a oportunidade de execução a outro thread, mesmo que os threads possuam a mesma prioridade. Se uma abordagem mais igualitária é empregada ou não, é uma decisão que cabe à implementação da máquina virtual Java. A lição correspondente é:

não suponha uma abordagem específica, uma ordem específica de execução
dos threads para o comportamento de um programa concorrente em Java.
