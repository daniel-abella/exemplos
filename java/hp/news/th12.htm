Estados de um thread

Um thread passa por vários estados. Não estamos interessados em filosofar sobre Java e, portanto, o que se passa antes da criação de um thread e o que se passa depois que este é destruído não é assunto tratado neste texto. O que interessa são os estados entre estes dois extremos. E eles são cinco: (1) novo; (2) executável; (3) executando; (4) esperando ou bloqueado e (5) morto.

O estado novo é obtido logo após uma instância de Thread ser criada, mas antes que o método start() seja chamado. Noutras palavras, trata-se apenas de um objeto Thread e não um thread em execução.

O estado executável é aquele atingido pelo thread quando este encontra-se pronto para execução. Um único processador pode executar vários threads concorrentemente. Em conseqüência, em determinado instante de tempo, vários threads podem estar "prontos" para execução, mas apenas um pode estar em execução. Aqueles que estão prontos aguardam pela oportunidade de serem executados, momento em que sai do estado executável para executando.

Um thread que se encontra em execução pode ter sua execução interrompida por vários motivos. Por exemplo, o tempo dedicado a execução do thread já foi todo consumido. Neste caso, o thread em questão retorna para o conjunto de threads executáveis, que ficam aguardando para serem executados. Em outros casos, o thread pode executar uma operação de espera como sleep() ou mesmo aguardar pelo término de outra operação. Nestes casos, o thread em questão sai do estado executando para o estado bloqueado.

Um thread fica no estado bloqueado até que a condição que o conduziu a este estado tenha sido eliminada. Quando isto ocorre, o thread não é conduzido novamente para execução, mas para o estado executável, juntamente com os demais que aguardam por uma oportunidade de usar o processador.
