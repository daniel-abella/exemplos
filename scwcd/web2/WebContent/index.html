Ilustra mecanismo de segurança via método FORM. Observe que o descritor da aplicação exige autenticação apenas para o método GET. Ou seja, o mesmo recurso que exige autenticação pelo método GET será acessado sem necessidade de autenticação pelo método POST. O acesso a recurso "inseguro" significa que não será exigida autenticação.
  • Seguro (GET). Exige autenticação.
  • POST não está configurado para exigir autenticação
  • Inseguro. Não exige autenticação.
