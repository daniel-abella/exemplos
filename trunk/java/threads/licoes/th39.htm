Sincronização de um bloco de código

A classe abaixo contém um método que não é declarado synchronized, contudo, parte deste método, um bloco de código, foi declarado synchronized sobre o objeto em questão. Noutras palavras, para um dado objeto desta classe, compartilhado entre dois ou mais threads, apenas um estará executando, por vez, o bloco de código synchronized. Observe que é possível a execução concorrente do bloco de código, desde que os threads façam uso de instâncias distintas da classe BlocoSincronizado.

O programa abaixo ilustra um bloco de código sincronizado. As duas linhas de código contidas neste bloco serão executadas, necessariamente, por um único thread por vez, para cada objeto desta classe. A sincronização é feita por objeto. Desta forma, se tivermos instâncias distintas de BlocoSincronizado, o bloco de código sincronizado poderá ser executado, concorrentemente, por mais de um thread, desde que estejam operando sobre objetos distintos.

public class BlocoSincronizado {
    public void tarefaOnerosa() {
        for (long i = 0; i < (long) 1E+12; i++) {
            synchronized (this) {
                double sqrt = Math.sqrt(i);
                assert Math.round(sqrt * sqrt) == i : i;
            }
         }
    }

    public static void main(String[] args) {
        BlocoSincronizado bs = new BlocoSincronizado();
        bs.tarefaOnerosa();
    }
}
