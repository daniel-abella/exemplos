Um bloco de código pode ser declarado synchronized

Embora seja freqüente a sincronização de todo um método, apenas parte de um método, ou bloco de código pode ser sincronizado. Em geral um bloco de código é sincronizado quando apenas parte de um método é necessária, ao contrário de todo o método. Isto tem implicações benéficas da perspectiva de desempenho.

Quando um método é sincronizado, o thread deve adquirir o lock de um objeto. Quando um bloco de código é sincronizado, o thread deverá adquirir o lock do objeto estabelecido quando o bloco de código sincronizado foi declarado. A conseqüência é que um mesmo objeto pode conter partes sincronizadas que dependem de mais de um lock.
