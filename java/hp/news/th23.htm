Quando se sai do estado executando?

Há vários cenários onde um thread interrompe a sua execução, ou seja, deixa o estado executando para algum outro estado. Um deles é quando o thread termina a execução do método run(). Neste caso, o thread simplesmente vai para o estado morto. Outros casos são comentados abaixo.

Um thread pode executar o método sleep(), o que o retira do estado executando pelo menos pelo intervalo especificado como argumento. Uma chamada ao método yield() oferece a oportunida para que outro thread de mesma prioridade possa ser executado, embora não se possa garantir este comportamento. Uma chamada ao método join() retira de execução o thread corrente até que aquele para o qual este método é invocado termine a execução do correspondente método run().

Dois outros cenários envolvem a sincronização entre threads: chamada ao método wait() e quando o thread não pode obter o lock de um objeto sobre o qual alguma operação deverá ser executada.
