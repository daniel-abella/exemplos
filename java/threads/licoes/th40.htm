Classe possui lock

Convém ressaltar que toda classe carregada pela máquina virtual Java possui uma instância correspondente da classe java.lang.Class que representa esta classe carregada. O lock da instância de java.lang.Class é empregado para sincronizar os elementos estáticos de uma classe.

É importante discernir entre elementos de classe e aqueles de instância. Aqueles de classe são declarados pela palavra reservada static. Estes podem ser utilizados sem que uma instância seja criada e, em conseqüência, a sincronização deles é baseada na instância de java.lang.Class correspondente à classe em questão. Ou seja, para todo método declarado synchronized e static, apenas um estará em execução por vez em uma dada máquina virtual Java, independente de quantos threads estejam interessados na execução deste método.
