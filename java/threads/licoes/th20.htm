Como um thread cede sua vez de execução?

Sabemos que em termos de programação concorrente há várias "tendências", mas nenhuma garantia. Por exemplo, um thread em execução pode "abrir mão" da sua vez de execução ao invocar o método yield(), que tira o thread do estado executando e passa-o para o estado executável.

Há uma tendência de que outro thread seja escolhido para execução, pois não há garantia de que o mesmo thread não seja o selecionado, mesmo que existam outros threads de mesma prioridade em execução. Esta tendência deve se confirmar em muitos casos, embora não seja uma garantia. O programa abaixo ilustra o emprego do método yield().

public class Yield implements Runnable {
    
    public void run() {
        for (int i = 0; i < 1000 * 1000; i++) {
            if (Thread.currentThread().getName().equals("1")) {
                if (i % 1000 == 0) {
                    Thread.yield();
                }
            }
        }
        System.out.println("FIM: " + Thread.currentThread().getName());
     }

    public static void main(String[] args) {
        Thread t1 = new Thread(new Yield(), "1");
        t1.start();

        // O intervalo abaixo dá oportunidade para o thread "1"
        // terminar antes do thread "2" (sem garantia!)
        try {
            Thread.sleep(100);
        } catch (InterrupedException ex) {
        }
 
        Thread t2 = new Thread(new Yield(), "2");
        t2.start();
    }
}
