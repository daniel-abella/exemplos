Concorrência sem sincronização

Quando memória é compartilhada entre threads, torna-se necessário sincronizar as operações entre eles. O programa ilustra a criação de dois threads que concorrentemente têm acesso à variável estática saldo. Cada um deles realiza 10000 subtrações concorrentes com as subtrações do outro thread. Aparentemente, o valor 20000 da variável compartilhada deveria ser reduzido a zero. Execuções sucessivas do programa ilustrado abaixo, contudo, irão mostrar que este será apenas um dos vários resultados possíveis. Nenhum valor resultante será negativo, mas muitos serão superiores a zero. Nestes casos, fica claro que entre a consulta do valor da variável e a correspondente subtração e atribuição do novo valor, um outro thread pode consultar o valor corrente da variável, o que resultaria em muitas subtrações sendo tratadas como apenas uma subtração. 

public class Sincronizar extends Thread {
	private static long saldo = 20000;
   public void debito(long v) {
       saldo = saldo - v; 
   }

   public void run() {
      for (int i = 0; i < 10000; i++)
          debito(1);
      }
   }
   public static void main(String[] args) throws Exception {
      Sincronizar thread1 = new Sincronizar();
      Sincronizar thread2 = new Sincronizar();
      thread1.start(); // Inicia thread (deveria debitar 10000)
      thread2.start(); // Inicia thread (deveria debitar 10000)
      thread1.join();
      thread2.join();
      // Saldo final deveria ser 20000
      System.out.println("Saldo final: " + saldo);
   }
}
