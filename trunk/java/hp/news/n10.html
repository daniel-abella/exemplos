Facilite a sua vida, crie algoritmos simples e organize o código!

Escrever código que seja "fácil" de ser alterado não é uma tarefa simples. Algumas dicas simples, se seguidas, podem produzir bons resultados. Vamos falar aqui de uma delas: mantenha coisas distintas isoladas. O que isto quer dizer?

Quando se pede para criar uma aplicação que exiba calendários, meses de um determinado ano, com a indicação dos dias da semana em que cada dia ocorre, por exemplo, uma divisão clara de tarefas existe entre código utilizado para apresentação e código do negócio. Código de apresentação é responsável pela interação com o usuário do computador. Apenas exibe dados e colhe as "intenções" dos usuários. Código do negócio refere-se à semântica da aplicação, à atividade em questão.

Por exemplo, código de apresentação para uma aplicação que exibe calendários é responsável por requisitar um ano cujo calendário correspondente deseja ser exibido no monitor e pela exibição do calendário. Por outro lado, código do negócio é aquele que reflete o conhecimento acerca da existência de anos bissextos, que contém o conhecimento de que o mês de outubro possui 31 dias e assim por diante.

Neste caso, a dica sugere que estes códigos devem ser mantidos isolados, por exemplo, em classes distintas, pois refletem conhecimentos distintos, com propósitos distintos e, portanto, o simples isolamento deles facilita a compreensão do código resultante e as conseqüentes alterações. Imprimir o mês de outubro, cujo primeiro dia se inicia numa quarta-feira e possui 31 dias pode ser tão simples quanto executar a classe abaixo. Desde, é claro, que exista uma classe responsável por produzir a saída, neste caso, oportunamente identificada por ImprimeCalendario. Uma classe com este nome sabe pouco sobre anos bissextos e o calendário gregoriano que empregamos, mas espera-se que saiba como imprimi-lo de forma tabular e elegante aos nossos olhos. Em tempo, a classe abaixo apenas testa a classe ImprimeCalendario, só serve a este propósito, conforme o nome TesteImprimeCalendario.

package calendar;

public class TesteImprimeCalendario {
    public static void main(String[] args) {
        ImprimeCalendario.imprimeMes(4,31);
    }
}

Quando isolamos estes conhecimentos, neste caso, o dia da semana em que o ocorre o primeiro dia de outubro de 2003 e quantos dias este mês possui, observe que estas informações foram fornecidas na chamada ao método imprimeMes, a tarefa de impressão fica simplificada. De fato tão simples quanto apresentado abaixo. É claro que podemos complicar com arrays, matrizes e outros, mas sabemos que este código será alterado e, portanto, devemos construí-lo de forma a facilitar as mudanças.

package calendar;
public class ImprimeCalendario {
    public static void imprimeDia(int dia) {
        System.out.print(((dia < 10) ? " " : " ") + dia + " ");
    }
    public static void imprimeSemana(int inicioDia, int fimDia) {
        for (int dia = inicioDia; dia <= fimDia; dia++) {
            imprimeDia(dia);
        }
        System.out.println();
    }
    public static void imprimeMes(int inicioSemana, int nDias) {
        System.out.println("DOM SEG TER QUA QUI SEX SAB");
        for (int i = (inicioSemana - 1) * 4; i > 0; i--) {
            System.out.print(" ");
        }
        int di = 1;
        int df = 7 - inicioSemana + 1;
        while (di <= nDias) {
            imprimeSemana(di, df);
            di = df + 1;
            df = di + (((di + 6) > nDias) ? (nDias - di) : 6);
        }
    }
}
