Lock é por objeto, não é por método

Um lock existe por objeto em Java. Cada objeto possui o seu próprio lock. Quando um thread adquire o lock do objeto para a execução de um método synchronized, nenhum outro thread poderá adquir este lock enquanto este não for liberado. Dado que só existe um por objeto, qualquer outro thread que tentar executar, qualquer que seja o método synchronized do mesmo objeto tornar-se-á bloqueado.

Se um thread tentar executar um método que não é declarado synchronized, então este será executado sem que o thread tenha que adquirir o lock do objeto em questão. O lock só é adquirido quando se tenta executar código synchronized. Se o código não é declarado synchronized, então não é preciso adquirir o lock.

Se um thread A envia ao objeto obj a mensagem a() (método declarado synchronized), então um thread B não poderá executar o método b() do mesmo objeto obj, caso o método b() seja declarado synchronized. A execução do método b() pelo thread B deverá aguardar pelo término da execução do método a(), pelo thread A, quando o lock do objeto obj será liberado. Convém ressaltar que a execução de b() só será iniciada após o thread B adquirir o lock do objeto obj.
