Todo objeto possui: lock e conjunto de threads em espera

Todo objeto possui um lock. Este lock deve ser adquirido de forma exclusiva por qualquer que seja o thread interessado em executar algum bloco de código sincronizado deste objeto. Também é possível estabelecer bloco de código sincronizado que depende do lock de outro objeto!

Todo objeto também possui um conjunto de threads em espera. Quando um thread executa o método wait para algum objeto, este thread libera o lock adquirido do objeto que recebe esta mensagem (um wait só pode ser executado de um bloco sincronizado do mesmo objeto) e interrompe a sua execução para aguardar por um evento no conjunto de threads associado ao objeto em questão. É algo como "olha, ficarei aguardando por algum sinal para eu voltar a executar novamente aqui na fila do objeto para o qual enviei a mensagem wait, tudo bem?" Este sinal vem na forma da mensagem notify() ou notifyAll().
