Sincronização exige o conceito de lock

Todo objeto em Java possui, intrinsecamente, um lock (ou tranca). Este lock é empregado sempre que se deseja a execução sincronizada de código. Ou seja, código declarado com a palavra reservada synchronized. Tal tipo de código só pode ser executado por um thread por vez e, em particular, após adquirir o lock do objeto em questão.

Um objeto, qualquer que seja este, possui um único lock. Para executar código sincronizado por um dado objeto é necessário adquirir o lock deste objeto. Se um thread obtém o lock deste objeto, então o código sincronizado poderá ser executado pelo thread que adquiriu o lock. Se o lock não está liberado (já foi adquirido), então o thread que tenta executar o código sincronizado pelo objeto permanece bloqueado até que o lock seja liberado.

O lock de um objeto é liberado quando o thread que executa o código sincronizado, ou seja, o thread que adquiriu o lock termina a execução do código sincronizado. O lock permanece liberado até que seja adquirido por um thread.
