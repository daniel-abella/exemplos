Adquirir lock é exigência exclusiva de código synchronized

Código que não é sincronizado, bem como campos de um objeto podem ser utilizados por um thread que não adquiriu o lock do objeto. De fato, algum thread pode estar executando um bloco sincronizado de um determinado objeto enquanto outros threads podem estar executando métodos não sincronizados e fazendo uso de campos deste mesmo objeto.
