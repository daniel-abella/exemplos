Concorrência

Programas são desenvolvidos, em geral, para serem executados seqüencialmente, uma instrução após a outra. Em um programa assim, qualquer que seja o instante de tempo considerado da execução dele, haverá uma única sentença em execução.

Alguns programas, contudo, permitem que o usuário continue interagindo com a interface gráfica, por exemplo, enquanto uma tarefa é executada concorrentemente. É possível até mesmo interromper esta tarefa executada concorrentemente com a interface gráfica. Nestes casos, em determinado instante de tempo, mais de uma instrução do mesmo programa encontra-se em execução de forma concorrente.

Não é necessário que as instruções em execução de um programa concorrente sejam executadas simultaneamente. Em uma máquina com uma única CPU, por exemplo, de fato, em um instante de tempo só uma instrução pode estar em execução. Outras, contudo, podem estar aguardando uma oportunidade de execução concorrente.
