Quando declarar um método synchronized?

Se há um único thread, então não existe a possibilidade de um mesmo método ou bloco de código ser executado concorrentemente. Apenas o thread existente terá acesso e, neste caso, exclusivo, a todo e qualquer método e/ou bloco de código.

Se há pelo menos dois threads, então é possível a execução concorrente de um mesmo método e/ou bloco de código. Sem perda de generalidade, assuma que o método m() será executado concorrente. Neste cenário, alguns contextos devem ser considerados.

Se o método m() faz uso exclusivo de memória local, então não há possibilidade de concorrência conduzir a um estado inválido, pois cada execução terá suas próprias variáveis locais, sem nenhuma ligação entre elas. A concorrência só pode trazer dificuldades quando recursos são compartilhados. Uma impressora, por exemplo, não admite concorrência durante a impressão de uma única página. De fato, todas as impressoras que conheço só admitem outra impressão após a anterior ser concluída por completo.

Memória também é um recurso compartilhado cujo acesso concorrente, quando não realizado de forma controlada, pode conduzir a estados inconsistentes. Se o método m() faz uso de memória compartilhada entre pelo menos dois threads, então será necessário sincronizar o acesso ao método.
