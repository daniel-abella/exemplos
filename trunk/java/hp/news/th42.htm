Deadlock

Assuma que o thread A contém código que tenta adquir dois locks, conforme ilustra o código abaixo. 

synchronized (objetoX) {
    synchronized (objetoY) {
        // Código que exige os locks de dois objetos
    }
}

Um outro thread, digamos B, realiza uma operação similar, conforme o código abaixo.

synchronized (objetoY) {
    synchronized (objetoX) {
        // Código que exige os locks de dois objetos
    }
}

Suponha o cenário onde o thread A adquiriu o lock do objeto objectoX e o thread B adquiriu o lock do objetoY. Neste contexto, qualquer que seja a ordem, um dos threads tentará adquirir o lock que já foi obtido pelo outro thread. Em conseqüência, ambos estarão bloqueados. O thread A aguardando pela liberação do lock do objeto objetoY e o thread B aguardando pelo lock do objeto objetoX. Contudo, nenhum destes locks será liberado. Ao contrário, os objetos que os retêm ficarão aguardando indefinidamente por uma liberação que jamais ocorrerá.
