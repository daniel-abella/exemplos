Métodos wait, notify e notifyAll

Assim como todo objeto possui um lock, também possui um conjunto de threads que aguardam por uma notificação deste objeto. Threads ficam bloqueados neste conjunto para um dado objeto quando enviam a mensagem wait() para este objeto. Neste momento, dado que esta chamada é executada em código sincronizado, o lock do objeto em questão é liberado.

Para que um thread aguardando na fila de um objeto possa retomar a execução é necessário que a mensagem notify ou notifyAll seja enviada ao objeto em questão. Esta mensagem "acorda" um ou todos os objetos que aguardam. Quando acordado, um thread vai para o estado executável e permanece lá até que o escalonador o escolha para execução. Infelizmente isto ainda não é suficiente, afinal, a mensagem wait só foi executada porque o thread se encontrava executando código sincronizado e, portanto, retinha o lock que deverá ser readquirido para que possa prosseguir a execução.
