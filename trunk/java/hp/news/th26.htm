Concorrência com sincronização

Quando memória é compartilhada entre threads, instância de Debito do programa abaixo, torna-se necessário sincronizar as operações entre eles. O programa ilustra dois threads que concorrentemente têm acesso à instância db. O acesso é sincronizado através da palavra reservada synchronized. Isto significa que, um único thread por vez tem acesso ao método synchronized public void debitar(double).

Em conseqüência, um único thread por vez estará consultando o valor de saldo por vez. Não interessa quantos sejam os threads tentando realizar uma operação de débito, um único thread estará realizando esta operação por vez.

class Debito implements Runnable {
    private double saldo = 10.0;

    synchronized public void debitar(double debito) {
        if (saldo >= debito) {
            Thread.yield();
            saldo = saldo - debito;
        }
    }

    public void run() {
        debitar(10);
    }

    public static void main(String[] args) throws InterruptedException {
        Debito db = new Debito();
        Thread t1 = new Thread(db);
        Thread t2 = new Thread(db);

        t1.start();
        t2.start();

        t1.join();
        t2.join();

        System.out.println(db.saldo);
    }
}
