Lock, wait e notify

Quando um thread executa o método wait(), ou uma das variantes deste, para um dado objeto, automaticamente o lock que foi adquirido para que tal execução fosse possível é liberado. O método notifyAll(), por outro lado, não apresenta este mesmo comportamento. Ou seja, quando um objeto recebe esta mensagem, todos os threads que aguardam por uma notificação deste objeto são "acordados", mas o lock do objeto correspondente não será liberado. Ou seja, só um dos threads acordados será executado, pois é preciso adquirir o lock que foi liberado anteriormente, no momento em que o método wait() foi executado.

O lock de um objeto que recebe a mensagem notifyAll(), só é liberado quando o bloco sincronizado onde esta mensagem é enviada for finalizado. O mesmo vale para o método notify().
