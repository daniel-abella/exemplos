Exception in thread "main" java.lang.NoClassDefFoundError

Quem nunca experimentou a exceção acima nunca programou em Java. O exemplo abaixo deve fazer com que esta exceção raramente volte a ocorrer.

A classe ClasseA, que não possui nenhum membro, é declarada abaixo.

public class ClasseA {}

Uma classe que faz uso da classe acima é fornecida abaixo. Observe que uma instância da classe acima é criada. 

public class Teste {
    public static void main(String[] args) {
        ClasseA ca = new ClasseA();
    }
}

Quando a MVJ recebe para execução a classe Teste (classe acima), esta máquina precisa conhecer onde encontrar as classes necessárias para execução da classe fornecida. Neste caso, é preciso que a MVJ tenha acesso ao arquivo ClasseA.class, que contém os bytecodes correspondentes ao conteúdo do arquivo ClasseA.java.

O local onde classes devem ser procuradas é fornecida através da variável CLASSPATH. Tanto a MVJ (programa java) quanto o compilador javac fazem uso desta variável de ambiente. O valor desta variável também pode ser fornecido através de argumento na linha de comandos.

O compilador precisa conhecer a declaração de classes empregadas pelo código que está sendo compilado. Afinal, se o compilador não conhece a declaração de uma classe X, como saber se esta classe X possui o método M? Se o compilador não puder encontrar a declaração de X, então algo como "cannot resolve symbol" será a mensagem gerada pelo compilador.

A MVJ precisa da implementação (bytecodes) de uma classe empregada para que esta possa ser executada. Afinal, se os bytecodes correspondentes a uma classe X não estiverem acessíveis, como executá-los? Se a MVJ não encontrar os bytecodes correspondentes à classe ClasseA, então ao executar a classe Teste irá gerar a seguinte exceção. 

Exception in thread "main" java.lang.NoClassDefFoundError: ClasseA
        at Teste.main(Teste.java:3)

Se você se deparar com a exceção acima, já sabe que a MVJ não encontra a classe indicada. Ou seja, não encontra o arquivo ClasseA.class, que implementa a classe ClasseA. Observe que o arquivo pode existir, pode estar inclusive no diretório corrente, contudo, é preciso fazer com que a MVJ também saiba, o que é realizado, novamente, através da variável CLASSPATH. Se os bytecodes (arquivo .class) existem e estão no diretório c:\myclasses, execute a MVJ da seguinte forma:

java -classpath c:\myclasses Teste

A sentença acima executa a MVJ para a classe Teste e, adicionalmente, diz o seguinte para esta MVJ: "olha, se você não encontrar alguma classe necessária para a execução da classe Teste, inclusive a própria, por favor, verifique se o que você precisa encontra-se no diretório c:\myclasses, pois é lá que deposito os arquivos .class". Você irá perceber que a MVJ, quando bem tratada, esforça-se ao máximo para nos servir!
