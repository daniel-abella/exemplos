Compilador javac e variável de ambiente CLASSPATH

O compilador javac e a máquina virtual Java java precisam de valor adequado na variável CLASSPATH para uma operação correta.

A variável CLASSPATH indica onde estas ferramentas, java e javac, devem procurar por classes (bytecodes). Estas classes podem estar contidas em diretórios ou arquivos .jar. Ou seja, o conteúdo da variável CLASSPATH é uma seqüência de diretórios e/ou arquivos .jar. No ambiente que utilizo, o conteúdo desta variável inclui um arquivo e um diretório, conforme abaixo:

c:\temp>echo %CLASSPATH%
c:\tools\j2ee\lib\j2ee.jar;c:\cloudscape_5.1\lib

Para ilustrar ilustrar o emprego desta variável de ambiente, observe o conteúdo do arquivo Teste.java, fornecido abaixo. 

public class Teste { 
    X objx; 
}

Quando se tenta compilar a classe acima, o resultado é aquele apresentado abaixo. 

c:\temp\javac Teste.java
Teste.java:2: cannot resolve symbol
symbol  : class X
location: class Teste
    X objx;
    ^
1 error

Isto porque o compilador não encontra a classe X, mesmo que esta classe esteja no mesmo diretório que a classe Teste. Observe que a variável CLASSPATH não contém o diretório corrente (.) ou explicitamente o diretório c:\temp, conforme ilustrado acima.

Há duas opções para indicar onde classes se encontram: (a) através da variável CLASSPATH ou (b) através de argumento fornecido na linha de comandos.

Uma descrição detalhada de como alterar esta variável para o ambiente Windows(r) pode ser encontrada aqui. Abaixo segue como fazer uso do argumento -classpath, ao executar o compilador javac.

c:\temp\javac -classpath . Teste.java

c:\temp>

Observe que o -classpath . diz ao compilador para procurar por classes no diretório corrente. Alternativamente poder-se-ia ter empregado -classpath c:\temp, com o mesmo efeito. Assumindo que a classe X.java esteja no mesmo diretório que Teste.java, o compilador compila o código deste arquivo sem erros.
