Teoria da criação de um thread

O primeiro passo da criação de um thread é estender a classe java.lang.Thread ou implementar a interface java.lang.Runnable. A tarefa a ser executada no thread a ser criado é fornecida pelo método public void run().

O segundo passo é criar uma instância da classe Thread. Não é possível criar um thread (execução concorrente) sem a existência de um objeto da classe java.lang.Thread. Há duas opções decorrentes da forma empregada para definir a atividade concorrente (passo anterior). Se foi empregada extensão da classe java.lang.Thread, então é suficiente criar uma instância desta classe. Caso contrário, será necessário criar uma instância de java.lang.Thread e, como argumento do construtor, fornecer um objeto que implementa java.lang.Runnable.

O terceiro e último passo, o ato principal, é enviar a mensagem start() para a instância da classe java.lang.Thread (passo anterior). É a execução do método start() que cria um thread. Se esta mensagem não é enviada, então nenhum thread é criado. Quando start() é enviada um novo thread é criado e passa a executar o que foi declarado no método run(). Tudo isto de forma concorrente com o thread que estava em execução, ou seja, aquele que criou uma instância da classe derivada e enviou a mensagem start().
