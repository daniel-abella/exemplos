Concorrência sem sincronização

Quando memória é compartilhada entre threads, instância de Debito do programa abaixo, torna-se necessário sincronizar as operações entre eles. O programa ilustra dois threads que concorrentemente têm acesso à instância db. O acesso não é sincronizado e, neste caso, é possível, após o início de uma operação de débito e antes do término desta, que outro thread consulte o valor de saldo, o que é um erro pois, neste momento, o valor consultado não é "real", pois uma operação de débito encontra-se em execução. O valor final de saldo mais provável é -10.

class Debito implements Runnable {
    private double saldo = 10.0;

    public void debitar(double debito) {
        if (saldo >= debito) {
            Thread.yield();
            saldo = saldo - debito;
        }
    }

    public void run() {
        debitar(10);
    }

    public static void main(String[] args) throws InterruptedException {
        Debito db = new Debito();
        Thread t1 = new Thread(db);
        Thread t2 = new Thread(db);

        t1.start();
        t2.start();

        t1.join();
        t2.join();

        System.out.println(db.saldo);
    }
}
