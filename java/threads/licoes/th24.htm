Por que é necessário sincronizar?

Quando dois ou mais threads compartilham memória, é necessário sincronizar as operações que manipulam esta memória. A palavra reservada synchronized é empregada para esta finalidade. O exemplo abaixo ilustra os efeitos indesejáveis quando a sincronização entre as operações não é realizada.

Sejam A e B threads que compartilham uma variável identificada por saldo, cujo valor inicial é 10. A e B têm como propósito realizar um débito de 10 sobre este valor, desde que haja saldo. O teste é fornecido abaixo.

if (saldo >= debito) {
    saldo = saldo - debito;
}

Suponha que o thread A tenha realizado o teste e se encontra pronta para a execução do débito propriamente dito. Neste momento o thread A pode ser bloqueado e o thread B conduzido à execução. O teste executado pelo thread B também será verdadeiro, pois o débito ainda não foi realizado pelo thread A. Neste momento tem-se dois threads que irão debitar, cada um, 10 do valor na variável saldo.

Independente da ordem em que estes débitos sejam realizados, o valor final na variável saldo será -10. Naturalmente, -10 é um cenário inconsistente para o valor de saldo, principalmente quando se observa que é feita uma verificação antes que o valor desta variável seja alterado. Este é um exemplo onde é necessário sincronizar a execução dos threads.
