Java, bytecodes e MVJ

A linguagem de programação Java não é passível de ser executada diretamente por processadores como o Intel(tm), por exemplo, que só é capaz de compreender o conjunto de instruções de máquina deste processador.

Para que este processador e outros possam executar programas em Java, é necessário a existência de um processador virtual. Ou seja, um processador que de fato não é real no sentido em que não existem componentes físicos. Este processador virtual é conhecido por Máquina Virtual Java ou MVJ. A MVJ é um programa (software). Portanto, não é físico como o Intel(tm), Motorola(tm) e outros, para citar alguns.

Um programa em Java primeiro é compilado, via o programa javac, para código em bytecodes. Bytecodes são instruções de máquina para a MVJ. Noutras palavras, a MVJ só é capaz de executar bytecodes, assim como processadores são capazes de executar exclusivamente as instruções da linguagem de máquina correspondente. Ou seja, qualquer que seja a plataforma considerada, se para esta é desenvolvida uma MVJ (um software), então nesta plataforma será possível executar código em Java, ou melhor, bytecodes!

Executar uma classe em Linux e executar o mesmo arquivo .class em Windows(r) produzem resultados semelhantes porque, em ambos os casos, quem está executando alguma coisa é a MVJ e o que está sendo executado são os mesmos bytecodes.

Teoricamente, código em Visual Basic, C, C++, Pascal ou outra linguagem de programação pode se beneficiar da mesma portabilidade de Java. Neste caso, bastaria que o compilador destas linguagens gerassem como resultado código em bytecodes. A MVJ apenas executa os bytecodes. Estes bytecodes não necessariamente devem ser gerados a partir de código em Java.
