Abordagem de construção de programas

Existem várias abordagens empregadas para se criar programas. Uma delas é atribuída a Dijkstra e data de 1968. Segundo ele, conforme esta abordagem, programas deveriam ser escritos em "níveis de abstrações".

Nestes níveis, um nível superior faz uso de serviços oferecidos pelo nível imediatamente inferior, ao passo que um nível inferior não pode fazer uso dos serviços oferecidos pelos níveis superiores. Nem faria sentido, pois, nestes casos, não teríamos os "níveis de abstrações".

Esta foi a abordagem empregada na classe ImprimeCalendario, fornecida abaixo. Esta classe é responsável pela exibição de um calendário com uma tabulação "agradável" aos olhares humanos.

Observe que esta classe possui três métodos, cada um em um nível de abstração. O método ImprimeMes faz uso do método ImprimeSemana, afinal um mês é uma composição de semanas. As semanas são formadas por dias e, daí, o método ImprimeSemana faz uso do método ImprimeDia. Observe que ImprimeDia não faz uso dos métodos nos outros níveis, afinal, porque um método encarregado de imprimir um determinado dia estaria fazendo uso de um método que imprime toda uma semana ou todo um mês? É claro que, neste caso, não teríamos uma organização apropriada do código. 

package calendar;
public class ImprimeCalendario {
    public static void imprimeDia(int dia) {
        System.out.print(((dia < 10) ? " " : " ") + dia + " ");
    }
    public static void imprimeSemana(int inicioDia, int fimDia) {
        for (int dia = inicioDia; dia <= fimDia; dia++) {
            imprimeDia(dia);
        }
        System.out.println();
    }
    public static void imprimeMes(int inicioSemana, int nDias) {
        System.out.println("DOM SEG TER QUA QUI SEX SAB");
        for (int i = (inicioSemana - 1) * 4; i > 0; i--) {
            System.out.print(" ");
        }
        int di = 1;
        int df = 7 - inicioSemana + 1;
        while (di <= nDias) {
            imprimeSemana(di, df);
            di = df + 1;
            df = di + (((di + 6) > nDias) ? (nDias - di) : 6);
        }
    }
}

Na próxima ocasião em que não souber como organizar a solução de um problema, tente a abordagem apresentada por Dijkstra, ou seja, tente organizar a sua solução em "níveis de abstração" distintos, como fizemos no exemplo acima: precisamos de imprimir um dia para que toda uma semana seja impressa e, naturalmente, precisamos de imprimir semanas para que um mês seja impresso.
