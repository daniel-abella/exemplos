Thread que requisita lock já adquirido é bloqueado

Se um thread tenta adquirir o lock de um objeto e este lock já foi adquirido por outro thread, então o thread que realiza a tentativa sai do estado executando e vai para o estado bloqueado.

O thread permanece bloqueado no lock do objeto em questão. Em um determinado instante pode existir todo um conjunto de threads bloqueados no lock de um determinado objeto. Quando o lock do objeto é liberado, não há nenhuma ordem específica empregada para determinar qual o próximo thread irá adquirir o lock que acabou de ser liberado. Novamente, como em muitos outros casos associados à programação concorrente em Java, nenhuma ordem, nenhum algoritmo específico é garantido para se estabelecer o próximo thread a adquirir o thread.
